# Deployment Guide (Render & GCP Cloud Run)

This guide walks you through deploying the application stack. We recommend using **Render.com** for a 100% free deployment that does not require a credit card, but also include instructions for **Google Cloud Run (GCP)**.

---

## Option 1: Free Deployment on Render (Recommended)

Render offers free web service hosting for Docker containers and static site builds without requiring any billing profiles.

### Step-by-Step Render Setup

1. **Push your code to GitHub**: Create a repository and push your codebase there. (The `.gitignore` will ensure your private credentials/local `.env` keys are not pushed).
2. **Go to [Render.com](https://render.com/)** and log in.
3. **Use the `render.yaml` file**:
   - In the Render Dashboard, click **Blueprints** -> **New Blueprint Instance**.
   - Connect your GitHub repository.
   - Render will read the `render.yaml` template file from your repository and prepare the 3 services (`lifesaver-ai-service`, `lifesaver-backend`, and `lifesaver-frontend`).
4. **Configure Environment Variables**:
   - In the Blueprint setup page, enter:
     - `GROQ_API_KEY`: Your Groq API key (e.g., `gsk_...`).
     - `MONGO_URI`: Your MongoDB Atlas database URL (e.g., `mongodb+srv://...`).
5. **Click Approve/Deploy**: Render will automatically build the containers in the background and launch them completely for free!

## Option 2: GCP Deployment (Google Cloud Run)

This guide walks you through containerizing and deploying the application services to **Google Cloud Run**. Note that GCP requires a linked billing account.

### Architecture & Services
1. **`ai-service`** (FastAPI) -> Deployed to Cloud Run.
2. **`server`** (Node/Express) -> Deployed to Cloud Run (points to `ai-service` URL).
3. **`client`** (React/Vite served via Nginx) -> Deployed to Cloud Run (configured with build argument `VITE_API_URL` pointing to Node server URL).

---

### Prerequisites
1. Install [Google Cloud SDK](https://cloud.google.com/sdk/docs/install).
2. Authenticate with Google Cloud:
   ```bash
   gcloud auth login
   ```
3. Create or select your GCP Project:
   ```bash
   gcloud config set project YOUR_PROJECT_ID
   ```
4. Enable the necessary APIs (Cloud Build, Cloud Run, Artifact Registry):
   ```bash
   gcloud services enable run.googleapis.com build.googleapis.com artifactregistry.googleapis.com
   ```

## 1. Environment & Security Configuration (.gitignore)
To prevent accidental exposure of passwords, API keys, and databases, the project is configured with a root-level `.gitignore` which ignores all `.env` files. 

> [!WARNING]
> Never check in or upload `.env` files to source control or GCP. Instead, configure environment variables directly on the Google Cloud Console or via the CLI flags shown below.

## 2. Step-by-Step Deployment

### Step A: Deploy the AI Service (FastAPI)
1. Build and push the image to Google Container/Artifact Registry:
   ```bash
   gcloud builds submit --tag gcr.io/YOUR_PROJECT_ID/ai-service ./ai-service
   ```
2. Deploy to Cloud Run (injecting `GROQ_API_KEY` from environment variables):
   ```bash
   gcloud run deploy ai-service \
     --image gcr.io/YOUR_PROJECT_ID/ai-service \
     --platform managed \
     --region us-central1 \
     --allow-unauthenticated \
     --set-env-vars GROQ_API_KEY="your_groq_api_key_here"
   ```
3. Copy the URL generated by Cloud Run (e.g., `https://ai-service-xxxx-uc.a.run.app`). You will need this for the backend deployment.

---

### Step B: Deploy the Backend Server (Express)
1. Build and push the Express server image:
   ```bash
   gcloud builds submit --tag gcr.io/YOUR_PROJECT_ID/server ./server
   ```
2. Deploy to Cloud Run, injecting MongoDB URI, JWT Secret, and the `AI_SERVICE_URL` pointing to the FastAPI URL from Step A:
   ```bash
   gcloud run deploy lifesaver-server \
     --image gcr.io/YOUR_PROJECT_ID/server \
     --platform managed \
     --region us-central1 \
     --allow-unauthenticated \
     --set-env-vars MONGO_URI="mongodb+srv://user:pass@cluster.mongodb.net/dbname",JWT_SECRET="your_jwt_secret_here",AI_SERVICE_URL="https://ai-service-xxxx-uc.a.run.app"
   ```
3. Copy the URL generated by Cloud Run (e.g., `https://lifesaver-server-xxxx-uc.a.run.app`). You will need this to build the client frontend.

---

### Step C: Deploy the Client Frontend (React/Vite)
Vite builds environment variables statically at compile-time. We configure the API URL using the Docker build argument `VITE_API_URL`, passing the backend server URL from Step B.

1. Build and push the client image, baking in the backend server's API URL:
   ```bash
   gcloud builds submit \
     --tag gcr.io/YOUR_PROJECT_ID/client \
     --build-arg VITE_API_URL="https://lifesaver-server-xxxx-uc.a.run.app/api" \
     ./client
   ```
2. Deploy the client container to Cloud Run (serving on standard container port 8080):
   ```bash
   gcloud run deploy lifesaver-client \
     --image gcr.io/YOUR_PROJECT_ID/client \
     --platform managed \
     --region us-central1 \
     --allow-unauthenticated
   ```
3. Cloud Run will output the final URL of your web application (e.g., `https://lifesaver-client-xxxx-uc.a.run.app`).

## 3. Best Practices (Secrets Manager)
For production, rather than passing raw keys (like `GROQ_API_KEY` or `MONGO_URI`) in plain-text environment variables, use **Google Secret Manager**:
1. Store the secret in Secret Manager:
   ```bash
   echo -n "gsk_..." | gcloud secrets create groq-api-key --data-file=-
   ```
2. Reference the secret during Cloud Run deployment:
   ```bash
   gcloud run deploy ai-service \
     --image gcr.io/YOUR_PROJECT_ID/ai-service \
     --update-secrets=GROQ_API_KEY=groq-api-key:latest \
     --platform managed \
     --region us-central1
   ```
